# Выводы по экспериментам 2 и 3 (matrix_lab)

## Эксперимент 2: Зависимость скорости от размерности матрицы

На малых размерах матрицы параллельное умножение может работать медленнее последовательного из‑за накладных расходов на создание процессов и передачу данных. С ростом размерности объём вычислений растёт быстрее (сложность O(n³)), и параллельная версия начинает давать заметное ускорение — выгоднее использовать её для больших матриц.

## Эксперимент 3: Сравнение вариантов линеаризации циклов

Порядок обхода индексов (i,k,j, k,i,j, j,i,k и т.д.) влияет на кэш-локальность и время выполнения. Разница между вариантами может достигать десятков процентов. Для каждого размера матрицы по результатам замеров выбирается оптимальный по времени вариант линеаризации.
